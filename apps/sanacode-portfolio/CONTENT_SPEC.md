# CONTENT_SPEC.md — Copy and Content Direction

## 1. Voice

The voice is:

- calm
- precise
- short
- reflective
- practical
- slightly poetic, but not vague

The voice is not:

- salesy
- corporate
- motivational
- overly technical
- generic developer profile

## 10. Words To Avoid

Avoid these unless necessary:

```txt
DX
革新的
最先端
誰でも簡単
業界初
効率化を実現
圧倒的
爆速
AIで全部解決
```

They make the site feel like a sales LP.

---

## 11. Content Density Rules

Per section:

- one main title
- one short subtitle
- one paragraph max
- one visual focus

Never write long explanatory sections.

If more detail is needed, link to project detail pages later.
